## ✅ PRINCÍPIOS ESTABELECIDOS

1. Nunca commitar direto em main ou dev
   └─ Sempre: feat/xxx → PR (base dev) → Wagner aprova → merge

2. Documentação viva em sincronismo contínuo
   └─ tarefas-log, tarefas-retomada, tarefas atualizam ao fim de cada sessão

3. Local = Remote sempre
   └─ Após cada PR mergeada: sincronizar, deletar branch, prune

4. Stack canônico não se negotia
   └─ Consultar mapa-executivo-plataforma ANTES de propor nova tech

5. Verificações obrigatórias após operações Git críticas
   └─ Nunca assumir que funcionou — sempre verificar
